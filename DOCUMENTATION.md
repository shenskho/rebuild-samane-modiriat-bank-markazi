# مستندات پروژه

## مقدمه

به پروژه خوش آمدید! این سند شما را با ساختار این برنامه آشنا می‌کند، مفاهیم اصلی را توضیح می‌دهد و به شما کمک می‌کند تا توسعه را شروع کنید. این پروژه یک داشبورد مدیریتی قدرتمند و انعطاف‌پذیر است که با React ساخته شده است.

## تکنولوژی‌های اصلی

این پروژه بر اساس یک پشته فناوری مدرن ساخته شده است. در اینجا تکنولوژی‌های کلیدی که باید با آن‌ها آشنا باشید، آورده شده است:

*   **React:** یک کتابخانه جاوا اسکریپт برای ساخت رابط‌های کاربری. تمام کامپوننت‌ها با استفاده از React ساخته شده‌اند.
*   **Vite:** یک ابزار ساخت سریع که تجربه توسعه سریع‌تر و سبک‌تری را برای پروژه‌های وب مدرن فراهم می‌کند.
*   **Redux:** یک نگهدارنده وضعیت قابل پیش‌بینی برای برنامه‌های جاوا اسکریپт. ما از آن برای مدیریت وضعیت سراسری برنامه استفاده می‌کنیم.
*   **React Router:** برای مدیریت ناوبری و مسیریابی در داخل برنامه.
*   **Reactstrap/Bootstrap:** برای کامپوننت‌های UI و استایل‌دهی.
*   **Sass:** یک پیش‌پردازنده CSS که قدرت و زیبایی را به زبان اصلی اضافه می‌کند.

## ساختار پوشه‌ها در سطح بالا

کد اصلی برنامه در پوشه `src` قرار دارد. در اینجا یک نمای کلی از ساختار آن آورده شده است:

*   `src/`: اینجا جایی است که تمام کدهای منبع برنامه زندگی می‌کنند.
    *   `@core/`: شامل کامپوننت‌ها، هوک‌ها و ابزارهای اصلی است که در سراسر برنامه به اشتراک گذاشته شده‌اند. این را به عنوان پایه و اساس UI در نظر بگیرید.
    *   `api/`: تمام درخواست‌های API به سرور را مدیریت می‌کند.
    *   `assets/`: تمام دارایی‌های استاتیک مانند تصاویر، فونت‌ها و شیوه‌نامه‌ها را در خود جای داده است.
    *   `components/`: شامل کامپوننت‌های UI قابل استفاده مجدد است که در سراسر برنامه استفاده می‌شوند.
    *   `configs/`: فایل‌های پیکربندی برای بخش‌های مختلف برنامه، مانند منوهای ناوبری.
    *   `contexts/`: Contextهای React را برای وضعیتی که نیازی به بودن در Redux ندارد، مدیریت می‌کند.
    *   `hooks/`: شامل هوک‌های سفارشی React است.
    *   `layouts/`: طرح‌بندی‌های مختلف برنامه را تعریف می‌کند (به عنوان مثال، منوی عمودی، منوی افقی، طرح‌بندی خالی).
    *   `navigation/`: ساختار منوی ناوبری را تعریف می‌کند.
    *   `router/`: شامل منطق مسیریابی برنامه است.
    *   `store/`: تمام کدهای مربوط به Redux، از جمله اکشن‌ها، ردیوسرها و خود استور را در خود جای داده است.
    *   `utility/`: مکانی برای توابع کمکی و کلاس‌های ابزاری.
    *   `views/`: شامل صفحات یا نماهای اصلی برنامه است. هر نما ترکیبی از کامپوننت‌ها است.
    *   `index.js`: نقطه ورود برنامه.
    *   `App.js`: کامپوننت ریشه برنامه.

این ساختار به گونه‌ای طراحی شده است که ماژولار و مقیاس‌پذیر باشد و پیدا کردن و مدیریت کد را با رشد پروژه آسان می‌کند.

## بررسی دقیق بخش‌های کلیدی

### `@core`

پوشه `@core` قلب UI برنامه است. این شامل کامپوننت‌های قابل استفاده مجدد، هوک‌های سفارشی و توابع ابزاری است که در سراسر پروژه استفاده می‌شوند.

*   **کامپوننت‌ها:** شما عناصر UI رایج مانند دکمه‌های سفارشی، فیلدهای ورودی و کامپوننت‌های طرح‌بندی را در اینجا پیدا خواهید کرد. این‌ها به گونه‌ای طراحی شده‌اند که عمومی و قابل انطباق باشند.
*   **هوک‌ها:** هوک‌های سفارشی React که منطق حالت‌دار را کپسوله کرده و مجدداً استفاده می‌کنند.
*   **Utils:** توابع ابزاری که می‌توانند در هر جای برنامه استفاده شوند.

### `store`

این پوشه شامل تمام کدهای مربوط به Redux برای مدیریت وضعیت است.

*   **`rootReducer.js`**: این فایل تمام ردیوسرهای جداگانه را در یک ردیوسر ریشه واحد برای استور Redux ترکیب می‌کند.
*   **`[feature]/`**: هر ویژگی برنامه زیرپوشه خود را دارد (به عنوان مثال، `users`، `products`) که شامل ردیوسر و اکشن‌های خود است.

**مثال: اضافه کردن یک بخش وضعیت جدید برای "شمارنده"**

1.  **یک ردیوسر ایجاد کنید:** در داخل پوشه `store`، یک فایل جدید به نام `counter.js` ایجاد کنید:

    ```javascript
    // src/store/counter.js
    import { createSlice } from '@reduxjs/toolkit';

    export const counterSlice = createSlice({
      name: 'counter',
      initialState: {
        value: 0,
      },
      reducers: {
        increment: (state) => {
          state.value += 1;
        },
        decrement: (state) => {
          state.value -= 1;
        },
      },
    });

    export const { increment, decrement } = counterSlice.actions;

    export default counterSlice.reducer;
    ```

2.  **ردیوسر را به ردیوسر ریشه اضافه کنید:** `src/store/rootReducer.js` را باز کرده و ردیوسر شمارنده را اضافه کنید:

    ```javascript
    // src/store/rootReducer.js
    import { combineReducers } from 'redux';
    import counterReducer from './counter'; // وارد کردن ردیوسر جدید

    const rootReducer = combineReducers({
      // ... سایر ردیوسرها
      counter: counterReducer, // اضافه کردن ردیوسر جدید
    });

    export default rootReducer;
    ```

### `router`

این پوشه ناوبری برنامه را مدیریت می‌کند.

*   **`Router.js`**: این فایل اصلی است که تمام مسیرهای برنامه را تعریف می‌کند. از `react-router-dom` برای نگاشت URLها به کامپوننت‌های خاص استفاده می‌کند.

**مثال: اضافه کردن یک مسیر جدید**

برای اضافه کردن یک صفحه جدید، باید یک مسیر جدید به `src/router/Router.js` اضافه کنید.

```javascript
// src/router/Router.js
import { lazy } from 'react';
import { useRoutes } from 'react-router-dom';

// ... سایر واردات

const MyNewPage = lazy(() => import('../views/MyNewPage'));

const Router = () => {
  const routes = useRoutes([
    // ... سایر مسیرها
    {
      path: '/my-new-page',
      element: <MyNewPage />,
    },
  ]);

  return routes;
};

export default Router;
```

### `assets`

این پوشه شامل تمام دارایی‌های استاتیک پروژه است. مهمترین بخش پوشه `scss` است که استایل‌ها را در خود جای داده است.

*   **`scss/`**: این پوشه شامل فایل‌های Sass برای استایل‌دهی برنامه است. ساختار آن به گونه‌ای است که ماژولار و نگهداری آن آسان باشد.
    *   **`variables.scss`**: طرح رنگ، فونت‌ها و سایر متغیرهای طراحی را تعریف می‌کند.
    *   **`main.scss`**: نقطه ورود اصلی برای تمام فایل‌های Sass.

### `api`

این پوشه مسئول تمام ارتباطات با سرور است. ما از `axios` برای ایجاد درخواست‌های HTTP استفاده می‌کنیم.

*   **`apiClient.js`**: این فایل اغلب شامل نمونه اصلی `axios` با تنظیمات از پیش پیکربندی شده مانند URL پایه و هدرها است.

**مثال: اضافه کردن یک فراخوانی API جدید**

1.  **یک تابع API ایجاد کنید:** در پوشه `api`، ممکن است یک فایل برای یک ویژگی خاص مانند `users.js` داشته باشید.

    ```javascript
    // src/api/users.js
    import apiClient from './apiClient';

    export const fetchUsers = () => {
      return apiClient.get('/users');
    };
    ```
